# AI-ассистент: Финансовый аналитик

Этот проект представляет собой прототип AI-ассистента, который помогает анализировать финансовые данные и отвечать на вопросы о результатах деятельности компании. Прототип разработан в рамках тестового задания.

Ассистент умеет читать структурированные финансовые данные (в формате CSV), автоматически рассчитывать ключевые производные метрики (рост выручки, операционную маржу, чистую маржу) и генерировать содержательные выводы с помощью LLM (через OpenRouter API), не придумывая данные, которых нет.

## Инструкция по установке и запуску

### 1. Предварительные требования
- Python 3.10 или выше
- Установленный `git`

### 2. Установка
1. Клонируйте репозиторий:
   ```bash
   git clone <ваш_URL_репозитория>
   cd Test-for-Junior-AI-engineer
   ```
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/MacOS:
   source venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте файл `.env` в корневой директории проекта и добавьте ваш API ключ OpenRouter:
   ```env
   OPENROUTER_API_KEY=ваш_api_ключ_здесь
   OPENROUTER_MODEL=openai/gpt-3.5-turbo  # или другая модель по желанию
   ```

### 3. Запуск
Запустите веб-интерфейс с помощью Streamlit:
```bash
streamlit run app.py
```
После запуска откроется вкладка в браузере (по умолчанию `http://localhost:8501`) с интерфейсом чата и предпросмотром данных.

## Описание архитектуры

Проект использует Agentic подход (использование LLM в качестве агента, который может писать и выполнять код) для работы с табличными данными любого объема без превышения лимитов токенов окна контекста.

1. **Обработка данных (`data_processor.py`)**
   Отвечает за загрузку CSV-файлов, валидацию наличия необходимых колонок и автоматический векторизованный расчет метрик (Revenue Growth, Operating Margin, Net Margin) с использованием `pandas`.

2. **AI-агент (`ai_agent.py`)**
   Использует `create_pandas_dataframe_agent` из `langchain_experimental`. Агент получает схему данных (не передавая сами данные в промпт) и системный промпт с ролью "Старшего финансового аналитика". При получении запроса агент генерирует Python/Pandas код для анализа загруженного датасета `df`, выполняет его и на основе полученных результатов формирует ответ.

3. **Веб-интерфейс (`app.py`)**
   Реализован на `streamlit`. Позволяет загружать пользовательские CSV-файлы, отображает превью данных и предоставляет интерфейс чата с индикацией загрузки и историей сообщений.

## Примеры запросов и ответов

**Вопрос 1:**
> В каком году был самый быстрый рост выручки?

**Ответ 1:**
> Самый быстрый рост выручки был в 2006 году.

**Вопрос 2:**
> Как изменялась прибыльность компании со временем?

**Ответ 2:**
> Прибыльность компании, измеряемая показателем чистой маржи (net margin), показывает следующую динамику со временем:
> - В 2005 году чистая маржа составляла 25.00%.
> - В 2014 году чистая маржа выросла до 25.03%.
> - Максимальное значение чистой маржи было достигнуто в 2019 году и составило 25.09%.
> - В последний доступный год, 2024, чистая маржа снизилась до 24.99%.
> Таким образом, прибыльность компании колеблется в пределах от 24.50% до 25.09% за рассматриваемый период времени.

**Вопрос 3:**
> Объясните динамику операционной маржи компании.

**Ответ 3:**
> Операционная маржа компании варьировалась в диапазоне от 24.49% до 25.09% за последние 20 лет. На протяжении большей части времени операционная маржа оставалась стабильной в пределах 24.5% - 25.1%. Наибольшие колебания произошли в начале периода, но затем маржа стабилизировалась и оставалась на относительно постоянном уровне. Таким образом, можно сделать вывод, что операционная маржа компании демонстрирует стабильность и умеренные колебания за последние 20 лет.
